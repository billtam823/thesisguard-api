## Architecture

ThesisGuard is a stock investment thesis management API. The core workflow: a user adds a stock to their watchlist, generates an AI-powered buy thesis, saves news items, and triggers daily reviews where the AI assesses whether news materially changes the investment thesis. If it does, the system updates the stock's status and fires an alert.

### Package Layout

Each domain is fully self-contained (entity + repository + service + controller + DTOs):

- `stock/` — watchlist management. `Stock` is the root aggregate (ticker, exchange, status).
- `thesis/` — one `StockThesis` per stock (OneToOne). Generated by AI, editable by user.
- `news/` — `NewsItem` records (ManyToOne → Stock). Also handles live previews from OpenBB (no DB write): company news (`GET .../news/fetch`), SEC 8-K filings (`GET .../news/filings`), and SEC Form 4 insider trades (`GET .../news/insider`). The SEC previews use OpenBB's `sec` provider, map results into the same `FetchedNewsItemResponse` shape (source `SEC EDGAR`), and return empty for non-US-listed stocks (`Stock.isUsListed()`).
- `review/` — `DailyNewsReview` (ManyToOne → Stock) with child `NewsAnalysisItem` records per news item.
- `alert/` — `Alert` records triggered when a review elevates thesis change level.
- `ai/` — `AiClient` interface with two implementations (see below).
- `openbb/` — `OpenBbClient` wraps the OpenBB Platform REST API.
- `common/exception/` — `ApiException` hierarchy + `GlobalExceptionHandler`.
- `config/` — CORS config (allows `localhost:5173` for the Vite frontend).

### Entity Relationships

```
Stock ──1:1──> StockThesis
     ──1:N──> NewsItem
     ──1:N──> DailyNewsReview ──1:N──> NewsAnalysisItem ──N:1──> NewsItem
     ──1:N──> Alert ──N:1──> DailyNewsReview (optional)
```

All relationships use `FetchType.LAZY`. Cascades (`ALL` + `orphanRemoval`) flow from parent to child.

### AI Client Strategy

`AiClient` interface has two beans with conditional activation:

- `OpenRouterAiClient` — active when `openrouter.api-key` is set (`@ConditionalOnProperty`). Calls `https://openrouter.ai/api/v1` chat completions with two models: `openrouter.model` for thesis generation and `openrouter.review-model` for daily reviews. Prompts are loaded from `src/main/resources/prompts/`. Strips markdown fences from responses before JSON parsing.
- `MockAiClient` — active via `@ConditionalOnMissingBean(AiClient.class)` when no OpenRouter key. Keyword-based mock: news titles containing "broken/material/watch/minor" map to corresponding `ThesisChangeLevel` values for local testing.

### Stock Code Model

`Stock` carries three related fields:
- `ticker` — base symbol (e.g. `NVDA`, `RY`)
- `exchange` — friendly exchange name (e.g. `NASDAQ`, `TSX`); auto-populated from OpenBB equity profile on creation, or manually supplied
- `getProviderTicker()` — computed: US exchanges (NASDAQ, NYSE, AMEX, NYSE ARCA) return `ticker` as-is; others append `:EXCHANGE` (e.g. `RY:TSX`, `0700:HKEX`)

`StockResponse` exposes all three as `stock_code`, `exchange`, and `provider_ticker`.

### Path Variable Conventions

All stock-scoped endpoints use `{stockCode}` (not a numeric ID). `StockService.getEntity(String)` accepts either a numeric ID string or a ticker symbol and resolves both.

### Daily Review Workflow (`DailyNewsReviewService.reviewToday`)

1. Load stock entity + its saved thesis
2. Fetch today's `NewsItem` records for the stock
3. If no news → save review with `No_News_Found`, skip AI
4. If news → call `AiClient.reviewNews()` → save `DailyNewsReview` + `NewsAnalysisItem` records
5. Elevate `Stock.status` based on `ThesisChangeLevel`: `Watch_Change` → `Watch`, `Material_Change` → `Reduce_Review`, `Thesis_Broken` → `Sell_Review`
6. Create an `Alert` for `Watch_Change` and above

### Enum Serialization

`StockStatus` and `ThesisChangeLevel` use `@JsonValue`/`@JsonCreator` with human-readable labels (`"Hold"`, `"Watch Change"`, `"Thesis Broken"`, etc.) — not the enum constant names.

### Configuration

```yaml
# Key properties (application.yaml)
openbb.base-url: https://openbb.kingheung.com
openrouter.model: <model for thesis generation>
openrouter.review-model: <model for daily reviews>
openrouter.api-key: <in gitignored application-local.yaml at repo root; set to enable OpenRouterAiClient, omit to use MockAiClient>
```

Tests use `src/test/resources/application.yaml` which swaps the datasource to H2 (`MODE=PostgreSQL`) with `ddl-auto: create-drop`. No Docker needed for tests.

Schema is managed by Hibernate `ddl-auto: update` in production — no migration files.
